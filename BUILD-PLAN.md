# 最小切片构建清单 —— 让飞轮转第一圈

> 目标:把 **Node ②(拆层→IR)+ Node ⑤(修正捕获)** 跑通一圈,作为整个飞轮的第一寸动力。
> 不是做产品,是证明"死图进 → 可编辑资产出 → 修正回流 → 数据变厚"这个闭环能闭合。

## 0. 切片定义(锁死,别扩)

- **只吃一种图**:扁平设计的营销图(文字清晰、背景简单)。**不碰摄影合成海报**——那是冷启动杀手,留给 v2。
  - 注:`ir/example-fengyoujing-poster.ir.json` 是摄影合成的"靶子样例",用来压测 schema 表达力;**v1 的实际处理对象是更简单的扁平图**。
- **只做三类编辑**:改文案 / 本地化 / 改尺寸。
- **路由器写死单路径**,先不做图像分类。
- **Node ⑤ 捕获第一天就在**;Node ⑥ 重训可以手动批量,但捕获不能省。

## 1. 整片"算通过"的验收(Definition of Done)

一个人能在浏览器里完成这一整圈,且数据落地:

1. 上传一张扁平营销图 → 系统产出一份**通过 schema 校验**的 IR;
2. 编辑器渲染 IR,低置信元素被高亮(`needs_review`);
3. 改一句文案 / 换一种语言 / 导一个新尺寸 → 导出可用 PNG;
4. 上述每次改动 → 在 `corrections[]` 里生成一条合规的 `predicted→corrected`,并落进训练数据存储;
5. 跑一遍 benchmark 脚本 → 打印出"平均每图修正数"这个北极星指标的当前值。

**五条全绿 = 飞轮转了第一圈。**

## M0 · 地基(承重墙通电)

| # | 任务 | 谁 | 做到什么算完 |
|---|---|---|---|
| 0.1 | 冻结 IR v1 schema | 架构 | `ir-v1.schema.json` + 样例**双双通过** draft2020 校验(见 ir/README §6) |
| 0.2 | 仓库骨架 + CI | 后端 | `ingest/ editor/ capture/ bench/` 四目录;CI 跑 schema 校验 + lint |
| 0.3 | 测试图集 v0 | 数据 | 50 张扁平营销图,每张人工标"正确 IR"做 ground truth,固定不变(benchmark 基准) |

## M1 · Node ②:拆层 → IR(把图变成结构)

| # | 任务 | 谁 | 做到什么算完 |
|---|---|---|---|
| 1.1 | 元素检测 + 分割 | CV | VLM 检测 + SAM-2 切块,输出带 `bbox/type/z` 的元素草稿;50 张上元素召回率 ≥ 0.8 |
| 1.2 | 文字抽取 | CV | OCR(内容)+ 字体/字号/颜色识别,写入 `text` payload;`text.content` 字符准确率 ≥ 0.9 |
| 1.3 | 背景重建 | CV | 撕走前景后 inpaint 出干净背景层,写 `background`;先不追质量,能产出 + 带 `confidence` 即可 |
| 1.4 | 矢量/图标 | CV | 简单形状/分割线/图标 → `vector`(VTracer 或 path);能出 SVG 即可 |
| 1.5 | 组装 IR + 置信度 | 后端 | 拼成完整 IR;按阈值置 `needs_review`;**整份必过 schema 校验**(对接 0.1) |

> M1 出口:任意一张扁平营销图 → 一份合法、可渲染的 IR。

## M2 · 编辑器 + Node ⑤:改 + 捕获(闭合价值环 & 数据环)

| # | 任务 | 谁 | 做到什么算完 |
|---|---|---|---|
| 2.1 | 渲染 IR | 前端 | 按 `z` 叠放渲染,视觉接近原图;`needs_review` 元素高亮置顶 |
| 2.2 | 三类编辑 | 前端 | 只暴露 `editable[]` 字段:改 `text.content`、切 `lang`、改 `bbox`(resize) |
| 2.3 | **修正捕获** ★ | 前端+后端 | 每次编辑 diff 出 `predicted→corrected`,按 `field_path/kind` 生成合规 `Correction` 并落库;**这一步是整片的心脏** |
| 2.4 | 导出 | 前端 | IR → 拍平 PNG(多尺寸);导出物可直接用 |

> M2 出口:用户能改能导,且每一笔改动都变成了训练数据。

## M3 · 飞轮仪表盘(证明它在转,不是错觉)

| # | 任务 | 谁 | 做到什么算完 |
|---|---|---|---|
| 3.1 | 北极星指标 | 数据 | 在 50 张固定图集上算"平均每图修正数",出一个数;**这是判断飞轮是否转动的唯一硬指标** |
| 3.2 | 校验回环(轻量) | 后端 | IR 重渲染拍平 vs 原图做 SSIM/LPIPS,给整图一个还原分 |
| 3.3 | 训练数据导出 | 数据 | `corrections[]` 按 `kind` 分桶导出成训练集,供 Node ⑥ 手动重训用(本切片不自动重训) |

> M3 出口:能用一个数字回答"飞轮转了吗"。重训一轮后该数字应下降——那一刻闭环成立。

---

## 关键路径 & 风险

- **关键路径**:0.1 → 1.5 → 2.3。schema 不冻结,后面全是返工;2.3 不做,就只是工具不是飞轮。
- **冷启动**(头号风险):M1 出来必然糙,靠两点撑住留存——① 只挑最易拆的扁平图;② 对手是"PS 里重做几小时",哪怕拆七成要手修也比它快。
- **别做的事**:不要在本切片里追求拆层精度极致、不要上自动重训、不要扩图类型。**先让圈闭合,再让圈变好。**
